### 社内向けCMS作成 (2023/1 - 2023/3)

#### プロジェクト概要
自社プロダクトのサイトで、リスト表示されたリンク集の順番を変えたり、追加/削除ができるCMSを作成。

#### チーム情報
- チーム規模 : 2名
    - ディレクター 1
    - フロントエンドエンジニア 1
- 役割 : フロントエンド 兼 デザイン

#### 使用した技術
- HTML
- CSS (SCSS)
  - CSS設計：FLOCSS
- React 
  - TypeScript
- Vite
- AdobePhotoshop

#### 作業内容
- 要件定義から設計、デザイン、コーディングまでを行った。

#### 取り組んだ課題
- デザインは専門外だが、ユーザーにとって使いやすく、見やすいデザインを用意する必要があった。
- 使用技術は自由という案件だったので、個人開発はしていたが実務経験を積めずにいたReact / TypeScript / Viteを採用したが、慣れていない言語を使用することで開発スピードやコード品質の低下が懸念された。

#### 工夫したこと
- リストの順番を変えるのは上下左右のボタンを用意して操作できれば良いとのことだったが、ドラッグ&ドロップの方が直感的な操作ができると考え、スケジュールに余裕もあったので矢印とドラッグ&ドロップどちらでも操作できるように実装した。
- デザインスキルのキャッチアップを行い、グリッドレイアウトを採用したり視覚的にもわかりやすいデザインを目指して実装した。ある程度作業が進んだところでデザイナーに声をかけてフィードバックをもらい、デザインのブラッシュアップをしていった。
- 事前に使用技術の総予習をし、コーディングもChatGPTといったAiも活用することでスケジュールを遅延することなく開発を行った。

#### 取り組みの成果
- 直感的でわかりやすいとのフィードバックを多くもらえた。
- Aiの活用はコードのコメントや命名、処理の最適化といった可読性の面で大きく成果を上げ、Reactに詳しくない他のエンジニアでもCMSの改修が可能となった。
- これまでVanillaかjQueryでの開発が主だった社内でもReactを使った開発に前向きになり、私の作成した成果物が社内向けの教材の一つとなった。

### 電力情報サイト開発 (2022/8 - 2022/12)
#### プロジェクト概要
電力会社が発表した情報をもとに、全国の電力使用状況や、どこで停電が起きているかなどが一目でわかるWEBサイトを作成。

#### チーム情報
- チーム規模 : 4名
    - ディレクター 1
    - フロントエンドエンジニア 1
    - バックエンドエンジニア 1
    - デザイナー 1
- 役割 : フロントエンド

#### 使用した技術
- HTML
- CSS (SCSS)
  - CSS設計：FLOCSS
- JavaScript (Vanilla)
  - Leaflet
- gulp / webpack

#### 作業内容
- バックエンドから渡されるJSONを使用し、詳細設計からコーディングをした。

#### 取り組んだ課題
- 私自信、そして社内でもJavaScriptのライブラリであるLeafletを初めて使用し、その学習曲線に対処すること。
- サイトで使用するJSONが50を超える個数で非常に多く、これはパフォーマンスと読み込み速度に影響を及ぼす可能性があった。

#### 工夫したこと
- Leafletの使用は初めてだったが、公式ドキュメンテーションやコミュニティフォーラムを活用し、Leafletを習得した。
- 取り扱うJSONが全部で50ファイルを超える量であったため、読み込み速度を考慮した。具体的には、ページ読み込み時に全てのJSONを一度に取得するのではなく、ファーストビューに必要ないJSONはユーザーの操作によって必要になった時に読み込まれるように実装した。これにより、初期の読み込み時間を短縮し、ユーザー体験を向上させた。
- さらにブラウザのパフォーマンスを最適化するために、DOMへのアクセスを最小限にした。これにより、画面の更新が高速になり、全体的なウェブサイトのパフォーマンスも向上した。

#### 取り組みの成果
- 今後も別プロジェクトなどで積極的にLeafletを使用した開発を行なっていくことがきまり、その先駆けとなった。
- このプロジェクトが社内で表彰され、賞金を獲得した。

### キャンペーンLP制作 (2020/7 - 2020/9)

#### チーム情報
- チーム規模 : 5名
    - ディレクター 2
    - フロントエンドエンジニア 2
    - デザイナー 1
- 役割 : フロントエンド

#### 使用した技術
- HTML
- CSS (SCSS)
  - CSS設計：BEM
- Vue.js (Vue 2)
- webpack

#### 作業内容
- 詳細設計とコーディングをした。
- キャンペーン情報のJSONを読み込み、配列やDOM操作を中心に実装した。

#### 取り組んだ課題
- スケジュールが非常にタイトで、なおかつリリース日は厳守しなければいけなく、遅れが許されない状態でコーディングを進める必要があった。
- 全国で一斉にスタートするキャンペーンのため競合他社が多く、SEOやブラウザパフォーマンスを意識して検索上位に表示させる必要があった。

#### 特に工夫したこと
- 1人ではスケジュール通りの作業が困難だったため、もう一人のフロントエンドエンジニアに手伝ってもらい、分担をして作業を行った。各自実装できたところから相互レビューをし合い、スムーズに開発を進めることができた。
- SEOに何が有効なのかを間違ったSEO対策をしないためにも先入観をなくし1から学び直し、Vue.jsの実装ではブラウザへのDOMアクセスを最低限にしたり、複数ある手法の実行速度を比較し、速い手法を採用するなどしてパフォーマンスを良くすることを意識した。

#### 取り組んだ成果
- 無事にスケジュール通りにリリースすることができた。
- パフォーマンス測定ツール「Lighthouse」でもオールグリーンの高得点を獲得でき、検索結果でも上位に表示された。
